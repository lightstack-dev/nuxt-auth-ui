Configure the Nuxt Auth UI module to match your application's needs. The module provides sensible defaults while giving you full control over every aspect.

## Basic Configuration

Add the modules to your Nuxt configuration:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["@lightstack-dev/nuxt-auth-ui", "@logto/nuxt"],

  authUi: {
    // Your configuration here
  },
});
```

## Configuration Options

### Routes

Define custom paths for each authentication route. Each route can be configured independently, giving you complete control over your URL structure.

```typescript [nuxt.config.ts]
authUi: {
  routes: {
    signIn: '/auth/sign-in',      // Default
    signUp: '/auth/sign-up',      // Default
    signOut: '/auth/sign-out',    // Default
    profile: '/auth/profile',      // Default
  },
},
```

**Examples:**

```typescript [nuxt.config.ts]
// Simple custom paths
authUi: {
  routes: {
    signIn: '/login',
    signUp: '/register',
    signOut: '/logout',
    profile: '/account',
  },
},

// Nested paths
authUi: {
  routes: {
    signIn: '/account/login',
    profile: '/dashboard/profile',
  },
},
```

### Component Naming

Control the prefix for auto-imported components:

```typescript [nuxt.config.ts]
authUi: {
  componentPrefix: 'A',    // Default
},
```

This affects how components are named, for example:

- `A` → `<ASignInButton />`
- `Auth` → `<AuthSignInButton />`
- `My` → `<MySignInButton />`

### Redirects

Configure where users are sent after authentication actions:

```typescript [nuxt.config.ts]
authUi: {
  redirects: {
    afterSignIn: '/',     // Default
    afterSignOut: '/',    // Default
  },
},
```

**Examples:**

```typescript [nuxt.config.ts]
authUi: {
  redirects: {
    afterSignIn: '/dashboard',
    afterSignOut: '/goodbye',
  },
},
```

### Social Providers

Configure social sign-in providers:

```typescript [nuxt.config.ts]
authUi: {
  socialProviders: [
    {
      name: 'google',        // Provider identifier (used with Logto)
      label: 'Continue with Google', // Optional: Override default label
      icon: 'i-simple-icons-google', // Optional: Override default icon
      enabled: true          // Show/hide provider
    },
    {
      name: 'github',
      enabled: true          // Uses default label and icon
    },
    {
      name: 'microsoft',
      enabled: false         // Disabled
    }
  ]
},
```

**Note:** The `name` field must match the social connector identifier configured in your Logto instance. When users click a social provider button, they're redirected to Logto's OAuth flow using the `social:<provider>` direct sign-in format.

**Auto-detection:** If no providers are configured, the module will automatically detect enabled social connectors from your Logto instance. Configure providers explicitly to override auto-detection or customize labels and icons.

**Localization:** Provider labels are automatically localized using the `useAuthUILocale` composable. Common providers (Google, GitHub, Microsoft, Facebook, Apple, etc.) have built-in translations. You can override labels by setting the `label` field or through the i18n system.

### Middleware

Configure authentication middleware behavior:

```typescript [nuxt.config.ts]
authUi: {
  middleware: {
    global: false,    // Default: not applied globally
    name: 'auth',     // Default middleware name
  },
},
```

When `global: true`, the auth middleware is applied to all pages. Otherwise, you can apply it selectively:

```vue [page.vue]
<script setup>
definePageMeta({
  middleware: "auth",
});
</script>
```

### Messages

Customize text content throughout the auth pages and components:

```typescript [nuxt.config.ts]
authUi: {
  messages: {
    signIn: 'Log In',                                   // Button text
    signOut: 'Log Out',                                 // Sign out text
    signInTitle: 'Welcome back!',                       // Page title
    signInDescription: 'Please sign in to continue',    // Page description
  },
},
```

**For full internationalization**, use [`@nuxtjs/i18n`](https://i18n.nuxtjs.org/) alongside simple message overrides:

```typescript [nuxt.config.ts]
// Use @nuxtjs/i18n for app-wide localization
modules: ['@nuxtjs/i18n', '@lightstack-dev/nuxt-auth-ui'],

i18n: {
  locales: ['en', 'es', 'fr'],
  defaultLocale: 'en',
},

// Override auth messages per locale in your i18n files
// ~/locales/en.json: { "auth": { "signIn": "Sign In" } }
// ~/locales/es.json: { "auth": { "signIn": "Iniciar Sesión" } }
```

### Legal Documents

Configure URLs to link to legal documents using any keys and URLs you choose:

```typescript [nuxt.config.ts]
authUi: {
  legal: {
    termsOfService: '/terms',           // Standard key
    privacyPolicy: '/privacy',          // Standard key  
    cookiePolicy: '/cookies',           // Standard key
    userAgreement: '/user-agreement',   // Custom key
    dataProcessing: '/gdpr',            // Custom key
    // Add as many documents as needed with unique keys
  },
},
```

When configured, these documents appear as clickable links in the sign-up form with text indicating that sign-up implies consent. Components can control which documents to show:

```vue
<!-- Show all configured documents (default) -->
<ASignUpForm />

<!-- Show only specific documents using your configured keys -->
<ASignUpForm :legal="['termsOfService', 'privacyPolicy']" />
<ASignUpForm :legal="['userAgreement', 'dataProcessing']" />

<!-- Hide legal document links -->
<ASignUpForm :legal="false" />
```

**Flexibility:** You have complete control over document keys and URLs. Use standard keys like `termsOfService` or custom ones like `compliancePolicy` - the choice is yours.

**Type Safety:** At runtime, the `legal` prop type becomes more precise based on your configuration. If you configure `termsOfService` and `userAgreement`, the prop accepts `boolean | ('termsOfService' | 'userAgreement')[]`.

### Branding

Customize the appearance of authentication pages:

```typescript [nuxt.config.ts]
authUi: {
  appName: 'My Application',
  logo: '/logo.svg',
},
```

### Theming

Auth components automatically inherit your app's theme and styling from [Nuxt UI](https://ui4.nuxt.com/docs/getting-started/theme).

#### Icons

The module provides default icons for all authentication components, which you can customize through your app configuration. Icons follow Nuxt UI's standard pattern, prefixed with `auth`:

```typescript [app.config.ts]
ui: {
  icons: {
    authSignIn: 'i-lucide-log-in',                // Default
    authSignUp: 'i-lucide-user-plus',             // Default
    authSignOut: 'i-lucide-log-out',              // Default
    authProfile: 'i-lucide-user',                 // Default
    authUser: 'i-lucide-user-circle',             // Default
    authSettings: 'i-lucide-settings',            // Default
    authPassword: 'i-lucide-key',                 // Default
    authEmail: 'i-lucide-mail',                   // Default
    authGoogle: 'i-simple-icons-google',          // Default
    authGitHub: 'i-simple-icons-github',          // Default
    authMicrosoft: 'i-simple-icons-microsoft',    // Default
  },
},
```

Icons use the [Nuxt UI icon system](https://ui4.nuxt.com/docs/getting-started/icons/nuxt) which supports any Iconify icon (100,000+ icons) and custom SVG icons.

**Examples:**

```typescript [app.config.ts]
ui: {
  icons: {
    authSignIn: 'i-heroicons-arrow-right-on-rectangle',
    authSignUp: 'i-heroicons-user-plus',
    authSignOut: 'i-heroicons-arrow-left-on-rectangle',
    authProfile: 'i-heroicons-user-circle',
  },
},
```

## Complete Example

Here's a fully configured module showcasing all options:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["@lightstack-dev/nuxt-auth-ui", "@logto/nuxt"],

  authUi: {
    // Social providers
    socialProviders: [
      {
        name: "google",
        label: "Continue with Google",
        icon: "i-simple-icons-google",
        enabled: true,
      },
      {
        name: "github",
        label: "Continue with GitHub",
        icon: "i-simple-icons-github",
        enabled: true,
      },
    ],

    // Custom routes
    routes: {
      signIn: "/login",
      signUp: "/join",
      signOut: "/logout",
      profile: "/account/settings",
    },

    // Component prefix
    componentPrefix: "Auth",

    // Post-auth redirects
    redirects: {
      afterSignIn: "/dashboard",
      afterSignOut: "/goodbye",
    },

    // Branding
    appName: "My SaaS App",
    logo: "/images/logo.svg",

    // Legal documents
    legal: {
      termsOfService: "/terms",
      privacyPolicy: "/privacy",
    },

    // Middleware settings
    middleware: {
      global: false,
      name: "auth",
    },

    // Custom messages
    messages: {
      signIn: "Log In",
      signInTitle: "Welcome back!",
    },
  },
});
```

## Type Safety

The module is fully typed. You can import the configuration interface for type checking:

```typescript
import type { AuthUIConfig } from "@lightstack-dev/nuxt-auth-ui";

const config: AuthUIConfig = {
  routes: {
    signIn: "/login",
  },
};
```

## Runtime Access

Access the resolved configuration at runtime using the `useAuthUI` composable:

```vue
<script setup>
const auth = useAuthUI();

// Access routes
console.log(auth.getAuthUrl("sign-in")); // Returns configured signIn route
</script>
```
